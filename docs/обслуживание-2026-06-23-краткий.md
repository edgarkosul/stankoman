# Обслуживание сервера intertooler.ru — краткая сводка

**Дата:** 23 июня 2026 · **Сервер:** intertooler.ru (Ubuntu 24.04 LTS) · **Простой:** ~2 сек + кратковременная перезагрузка

## Что сделано
- ⬆️ **Обновлено ПО:** PHP 8.4.22, MariaDB 11.4.12, Docker 29.6 и 40 системных пакетов (вкл. обновления безопасности).
- 🔒 **Закрыто 27 уязвимостей** в зависимостях сайта (Laravel 12.62, Filament 5.6.7, Guzzle, Symfony) — протестировано и развёрнуто на проде.
- 🛠 **Устранены 3 скрытые проблемы:**
  1. Автопродление SSL-сертификата сайта (отказало бы в августе → сайт без HTTPS).
  2. Служба сертификатов «висела в сбое» из-за мёртвого домена-соседа.
  3. Сбой ночной очистки резервных копий.
- 🧹 Настроена ротация логов; проверены резервные копии и журналы входов (**следов взлома нет**).

## Состояние сейчас
Сайт работает (**HTTP/2 200**), сертификат — до 05.09.2026, все службы активны, **сбоев нет**, защита (firewall + fail2ban + автообновления) включена.

## Важные примечания
- **FTP без шифрования оставлен намеренно** — нужен для программы «Корсайт» (заливка файлов на kratonkuban.ru), которая не умеет защищённый протокол. Защищён fail2ban.
- **Панель ispmanager (порт 1500)** оставлена доступной — по согласованию.

## Рекомендации
- Регулярно обновлять зависимости (раз в 1–2 мес.).
- По возможности ограничить FTP/SSH/ispmanager по доверенным IP.
- Решить судьбу домена `stankoman.ru` (нет DNS-делегирования).
