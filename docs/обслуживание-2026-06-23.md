# Отчёт о техническом обслуживании сервера intertooler.ru

**Объект:** VDS `intertooler.ru` (188.120.237.93), Ubuntu 24.04 LTS
**Дата работ:** 23 июня 2026
**Тип:** плановое администрирование
**Суммарный простой:** ~2 секунды (перезапуск БД при обновлении) + кратковременная перезагрузка сервера в согласованное окно

## 1. Резюме
Проведено комплексное обслуживание сервера. Обновлено всё системное ПО и зависимости веб-приложения, **устранены три скрытые проблемы, которые в ближайшие недели привели бы к отказам** (продление SSL-сертификата сайта, служба сертификатов, очистка резервных копий), и **закрыты 27 уязвимостей** в зависимостях приложения. Признаков несанкционированного доступа не выявлено. Сайт работает штатно, изменения проверены на «живом» сервере.

## 3. Выполненные работы

### 3.1. Обновление системного ПО
40 пакетов, включая 4 обновления безопасности:
- PHP 8.4.18 → **8.4.22**
- MariaDB 11.4.10 → **11.4.12**
- Docker 29.3 → **29.6** (+ Compose)
- системные библиотеки, firmware (применены после перезагрузки)

### 3.2. Устранённые скрытые проблемы
1. **SSL-сертификат сайта** — автопродление было настроено на несуществующий путь; ближайшее продление (август) завершилось бы отказом, и сайт остался бы без HTTPS. Исправлено и проверено.
2. **Служба сертификатов** висела в состоянии «сбой» из-за домена-соседа `stankoman.ru`, у которого удалено DNS-делегирование. Нерабочие сертификаты убраны — служба снова исправна (домены `intertooler.ru` и `kratonkuban.ru` продлеваются штатно).
3. **Резервное копирование** — исправлен путь в плагине restic (ночная очистка копий завершалась с ошибкой); проверено.

### 3.3. Безопасность и обслуживание
- Настроена **ротация логов** приложения (файл рос неконтролируемо).
- Проверены журналы входов и fail2ban — **посторонних успешных входов нет**; брутфорс-попытки блокируются автоматически.
- Перед обновлениями сделан **страховочный дамп базы данных**.

### 3.4. Устранение уязвимостей в приложении
- По результатам `composer audit` обновлены 14 пакетов, **27 уязвимостей → 0** (в т.ч. критичные в Laravel и Symfony, CVE в Filament).
- Ключевое: Laravel 12.58 → **12.62**, Filament 5.6.2 → **5.6.7**, Guzzle 7.10 → **7.12.1**, Symfony-компоненты — до пропатченных версий.
- Изменения протестированы (700+ автотестов проходят) и **развёрнуты на проде** (релиз `20260623151824`).

## 4. Важные примечания по конфигурации (намеренные решения)

- **FTP без шифрования (порт 21) оставлен сознательно.** Он необходим для работы устаревшей программы **«Корсайт»**, которая заливает файлы на сайт **kratonkuban.ru** и не поддерживает защищённый протокол. Это осознанный компромисс. Текущие меры снижения риска: доступ по FTP защищён fail2ban (автоблокировка перебора паролей). **Рекомендация:** если «Корсайт» работает с фиксированного IP-адреса — ограничить FTP только этим адресом; в перспективе, при обновлении/замене «Корсайт», перейти на SFTP.
- **Панель управления ispmanager (порт 1500) оставлена доступной** — по согласованию. При желании в будущем доступ можно ограничить по списку IP.
